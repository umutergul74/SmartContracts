# Security policy

## Reporting this repository's vulnerabilities

Do not open a public issue for a vulnerability that could expose users or unpublished research.
Contact the repository owner privately through GitHub's security-advisory feature.

## Responsible target disclosure

Findings about third-party targets must be submitted only through that target's current authorized
channel. For the Arbitrum pilot, re-check the live Immunefi program immediately before submission.
Never publish an unpatched issue without explicit permission.

## Forbidden use

This project must not be used for:

- mainnet or public-testnet exploit attempts;
- transaction broadcasting;
- fund movement, draining, locking, or griefing;
- private-key, mnemonic, keystore, or wallet automation;
- off-chain infrastructure scanning without explicit authorization;
- significant traffic generation or denial of service;
- automatic bounty submission.

Local fixtures, isolated simulations, and read-only local forks are the supported PoC environments.
